# 🧪 QA Testing Portfolio (Manual Testing & Bug Reports)

This repository is a **Quality Assurance (QA) testing portfolio** that demonstrates hands-on experience in manual testing across **mobile (iOS & Android) and web applications**.

The goal of this portfolio is to showcase real-world QA skills including bug detection, exploratory testing, UI/UX evaluation, test case design, and structured reporting across different systems and environments.

---

## 🎯 Purpose

This portfolio was created to demonstrate practical QA knowledge and skills, including:

- Manual testing of mobile and web applications  
- Identification and documentation of software defects  
- UI/UX and usability evaluation  
- Exploratory testing across real user flows  
- Writing structured bug reports and test cases  
- Simulating real-world QA scenarios  
- Providing actionable feedback on product behavior  

---

## 📱 Projects Included

### 📱 1. ChillAround (iOS Mobile App Testing)

A social location-based iOS application focused on chat, geolocation, AR features, and city-based communities.

**Key features tested:**
- Chat system  
- Geolocation tracking  
- AR (Augmented Reality) features  
- Reality creation/joining system  
- Group creation and city communities  
- User profile management  
- In-app feedback system  

---

### 🤖 2. Memrly (Android Closed Beta Testing)

A private photo and video sharing Android application tested during a 14-day closed beta program.

**Key features tested:**
- Album creation and management  
- Photo and video uploads  
- Guest invitations  
- Album browsing and access control  
- Media performance and stability  
- Daily real-world usage testing  
- Crash and bug reporting  

---

### 🌐 3. Web App Testing (E-commerce Simulation)

A simulated e-commerce web application used for QA practice and functional testing scenarios.

**Key features tested:**
- Login and authentication  
- Product browsing and filtering  
- Cart functionality  
- Checkout flow  
- Form validation  
- UI responsiveness  
- Error handling scenarios

## 🐛 Bug Reporting Approach

All issues were documented using a structured QA format:

- Clear and concise bug title  
- Steps to reproduce  
- Expected vs actual behavior  
- Severity classification (Low / Medium / High / Critical)  
- Evidence (screenshots or recordings when available)  

---

## 📊 Example Bug Types Identified

Across different applications, examples of issues documented include:

- Chat message delays or failures  
- Geolocation inconsistencies  
- AR feature freezing or loading issues  
- Upload failures (images/videos)  
- Broken invite links or access issues  
- UI misalignment on mobile screens  
- Performance delays in loading content  
- Form validation issues in web applications
